## 认证预设

选择你的身份提供商。每个预设自动处理 issuer URL、JWKS 端点和密钥轮换：

## Scope 验证

工具可以声明所需的 scope。框架在 handler 执行前自动检查 token：

```typescript
server.registerTool(
  {
    name: "delete_user",
    description: "Delete a user account",
    inputSchema: {
      type: "object",
      properties: { userId: { type: "string" } },
      required: ["userId"],
    },
    requiredScopes: ["admin", "users:delete"],
  },
  async ({ userId }) => {
    // This code only runs if the token has BOTH admin AND users:delete
    await deleteUser(userId);
    return { deleted: true };
  },
);
```

<Aside>
  Scope 检查是**基于 AND 的** — token 必须拥有*所有*列出的 scope。需要 OR 逻辑？在[自定义中间件](/zh/docs/guides/middleware/)中实现。
</Aside>

## Token 缓存

`JwtAuthProvider` 内置内存缓存，避免重复 JWKS 查询：

- 缓存键：原始 token 字符串的 SHA-256 哈希
- TTL：5 分钟
- 最大条目：1000

这意味着 token 第一次针对 JWKS 端点验证，之后在 TTL 窗口内从缓存提供服务。

## 进阶：JwtAuthProvider

要完全控制验证参数，直接使用 `JwtAuthProvider`：

```typescript
import { JwtAuthProvider } from "@casys/mcp-server";

const provider = new JwtAuthProvider({
  issuer: "https://accounts.google.com",
  audience: "https://my-mcp.example.com",
  resource: "https://my-mcp.example.com",
  authorizationServers: ["https://accounts.google.com"],
  jwksUri: "https://www.googleapis.com/oauth2/v3/certs",
  scopesSupported: ["read", "write"],
});

const authInfo = await provider.verifyToken(token);
// AuthInfo | null
```

`AuthInfo` 字段：

```typescript
interface AuthInfo {
  subject: string;                     // Token "sub" claim
  clientId?: string;                   // Token "client_id" or "azp" claim
  scopes: string[];                    // Parsed from "scope" claim (space-separated)
  claims?: Record<string, unknown>;    // All token claims
  expiresAt?: number;                  // Token expiry as Unix epoch seconds
}
```

## 自动发现 (RFC 9728)

在 HTTP 服务器上配置认证后，框架自动提供发现端点。MCP 客户端用它来确定如何认证：

```
GET /.well-known/oauth-protected-resource
```

```json
{
  "resource": "https://my-mcp.example.com",
  "authorization_servers": ["https://accounts.google.com"],
  "scopes_supported": ["read", "write"],
  "bearer_methods_supported": ["header"]
}
```

<Aside type="tip">
  无需手动设置。这是从你的 auth provider 配置中派生的，自动提供服务。
</Aside>

## 客户端 OAuth 流程

构建需要对受保护服务器进行认证的 MCP 客户端：

```typescript
import {
  prepareOAuthProvider,
  FileTokenStore,
} from "@casys/mcp-server";

const oauthProvider = await prepareOAuthProvider(
  "https://my-mcp.example.com",
  {
    clientId: "my-client-id",
    scopes: ["read", "write"],
    tokenStore: new FileTokenStore("./.tokens"),
    openBrowser: (url) => {
      // Open the authorization URL in the user's browser
      console.log(`Open this URL to authorize: ${url}`);
    },
  },
);
```

提供了两种 token 存储实现：

- `FileTokenStore(path)` — 将 token 持久化到磁盘
- `MemoryTokenStore()` — 内存存储，重启后 token 丢失
