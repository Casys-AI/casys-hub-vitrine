MCP 平台由三个协同工作的软件包组成：**构建**带有 UI 生成工具的 MCP 服务器，**连接**到消息平台，以及将多个 UI **组合**成单一仪表板。

## 三个软件包

<CardGrid>
  <Card title="@casys/mcp-server" icon="rocket">
    基于 Deno 构建 MCP 服务器的框架。提供中间件管道、OAuth2、并发控制以及 `McpApp` — 用于注册工具并提供 `ui://` 资源的核心类。**v0.14.0，稳定版，301 个测试。**
  </Card>
  <Card title="@casys/mcp-bridge" icon="laptop">
    将 MCP 应用 UI 桥接到消息平台（Telegram、LINE）。运行 HTTP 资源服务器，通过 WebSocket JSON-RPC 将 HTML 页面嵌入平台 Webview。**v0.2.0，原型阶段。**
  </Card>
  <Card title="@casys/mcp-compose" icon="list-format">
    收集多个工具调用返回的 `_meta.ui` 结果，并将其组合成分栏 HTML 仪表板。内置事件总线，允许 iframe 之间通信。**v0.4.0，Alpha 阶段。**
  </Card>
</CardGrid>

## `_meta.ui` 约定

三个软件包通过一个共享约定相互连接：当工具返回结果时，包含一个 `_meta.ui.resourceUri` 字段。其值是一个 `ui://` URI，指向服务器上注册的 HTML 页面。

```typescript
// 携带 UI 资源的工具结果
{
  content: [{ type: "text", text: "Invoice INV-001 loaded" }],
  _meta: { ui: { resourceUri: "ui://invoices/invoice-viewer" } }
}
```

`mcp-server` 在该 URI 处注册 HTML。`mcp-bridge` 解析该 URI 并将 HTML 嵌入 Telegram Mini App Webview。`mcp-compose` 收集多个此类结果，并在 iframe 仪表板中并排渲染。

## 何时使用哪个软件包

**单独使用 `mcp-server`**：当你需要 MCP 服务器时 — 适用于任何工具构建场景。它不依赖 `mcp-bridge` 或 `mcp-compose`。

**添加 `mcp-bridge`**：当用户通过消息平台（Telegram、LINE）与服务器交互，且你希望在平台 Webview 中打开交互式 HTML 页面时。

**添加 `mcp-compose`**：当 LLM 工作流调用多个工具且每个工具都返回 UI，而你希望在单一仪表板中并排展示这些 UI 时。`mcp-compose` 可以在不使用 `mcp-bridge` 的情况下独立运行 — 它输出纯 HTML。

**同时使用三者**：当你需要完整的流水线时：带 UI 的 MCP 工具、通过消息平台分发、支持跨 UI 事件通信。
