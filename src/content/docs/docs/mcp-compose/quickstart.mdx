## Install

```bash
deno add jsr:@casys/mcp-compose@0.3.0
```

Or add the import manually:

```json
{
  "imports": {
    "@casys/mcp-compose": "jsr:@casys/mcp-compose@0.3.0"
  }
}
```

## Minimal pipeline

This example uses the same tool names and rule shape that appear in the repo README and tests:

```typescript
import {
  buildCompositeUi,
  createCollector,
  renderComposite,
  validateSyncRules,
} from "@casys/mcp-compose";

const collector = createCollector();

const pgResult = {
  content: [{ type: "text", text: "Query executed" }],
  _meta: { ui: { resourceUri: "ui://postgres/table/sales-q1" } },
};

const vizResult = {
  content: [{ type: "text", text: "Chart rendered" }],
  _meta: { ui: { resourceUri: "ui://viz/chart/sales-bar" } },
};

collector.collect("postgres:query", pgResult, {
  query: "SELECT * FROM sales WHERE quarter = 'Q1'",
  workflowId: "wf-001",
});
collector.collect("viz:render", vizResult, {
  workflowId: "wf-001",
});

const resources = collector.getResources();

const orchestration = {
  layout: "split" as const,
  sync: [
    { from: "postgres:query", event: "filter", to: "viz:render", action: "update" },
  ],
  sharedContext: ["workflowId"],
};

const validation = validateSyncRules(
  orchestration.sync,
  resources.map((resource) => resource.source),
);

if (!validation.valid) {
  console.error(validation.issues);
  Deno.exit(1);
}

const descriptor = buildCompositeUi(resources, orchestration);
const html = renderComposite(descriptor);

await Deno.writeTextFile("./dashboard.html", html);
```

## What you get

- Two collected resources with slots `0` and `1`.
- One composite descriptor whose sync rule is resolved to `{ from: 0, to: 1, ... }`.
- One HTML document with inline styles, two iframes, and the host event bus.

## What to remember

- `collect()` returns `null` for results that do not contain `_meta.ui.resourceUri`.
- `validateSyncRules()` is the step that surfaces sync problems. `buildCompositeUi()` does not throw on bad sync references.
- `renderComposite()` from the root package uses the host renderer, which supports the preset layouts and areas layouts documented on the [renderer](./renderer/) page.
