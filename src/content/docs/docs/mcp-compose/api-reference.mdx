These docs describe the API surface present in `mcp-compose` `0.3.0`.

## Root package

```typescript
import {
  buildCompositeUi,
  composeEvents,
  createCollector,
  createMcpSdkCollector,
  extractUiMeta,
  renderComposite,
  resolveSyncRules,
  uiMeta,
  validateComposition,
  validateSyncRules,
} from "@casys/mcp-compose";
```

Root exports include:

- core types such as `CollectedUiResource`, `UiLayout`, `UiOrchestration`, `UiSyncRule`, `CompositeUiDescriptor`
- `ErrorCode`, `UI_LAYOUT_PRESETS`, `isValidLayout`
- `createCollector`, `extractUiMeta`
- `resolveSyncRules`, `validateSyncRules`
- `buildCompositeUi`
- `renderComposite`
- SDK helpers: `createMcpSdkCollector`, `uiMeta`, `validateComposition`, `composeEvents`
- host types: `CompositeUiHost`, `HostConfig`

## `@casys/mcp-compose/core`

Use `core` when you only want the pure composition primitives:

```typescript
import {
  buildCompositeUi,
  createCollector,
  extractUiMeta,
  resolveSyncRules,
  validateSyncRules,
  ErrorCode,
  isLayoutAreas,
  isLayoutPreset,
  isValidLayout,
  UI_LAYOUT_PRESETS,
} from "@casys/mcp-compose/core";
```

`core` exports types, collector, sync helpers, composer, and layout validators. It does not export the renderer.

## `@casys/mcp-compose/sdk`

```typescript
import {
  composeEvents,
  createMcpSdkCollector,
  uiMeta,
  validateComposition,
  COMPOSE_EVENT_METHOD,
} from "@casys/mcp-compose/sdk";
```

Exports:

- `createMcpSdkCollector()`
- `uiMeta(options)`
- `validateComposition(tools, syncRules)`
- `composeEvents(parent?, source?)`
- `COMPOSE_EVENT_METHOD` — constant `"ui/compose/event"`
- related SDK types such as `McpSdkCallToolResult`, `UiMetaOptions`, `CompositionValidationResult`, `ComposeEvents`

Notes:

- `createMcpSdkCollector()` only adds SDK-shaped typing plus `isError: true` skipping.
- `composeEvents()` is browser-oriented. Creating the channel succeeds even without a parent window, but `emit()` throws if no parent is available.

## `@casys/mcp-compose/host`

```typescript
import {
  renderComposite,
  serveDashboard,
} from "@casys/mcp-compose/host";
```

Exports:

- `renderComposite(descriptor)`
- `serveDashboard(html, options?)`
- `CompositeUiHost`, `HostConfig`
- `ServeDashboardHandle`, `ServeDashboardOptions`

Notes:

- `renderComposite()` is pure.
- `serveDashboard()` is Deno-only. It uses `Deno.serve()` and may open the default browser.

## `@casys/mcp-compose/runtime`

```typescript
import {
  composeDashboard,
  composeDashboardFromFiles,
  connectHttp,
  createCluster,
  injectArgs,
  loadManifest,
  loadManifests,
  loadTemplate,
  parseManifest,
  parseTemplate,
  startServer,
  validateManifest,
  validateTemplate,
} from "@casys/mcp-compose/runtime";
```

Exports:

- manifest helpers: `validateManifest`, `parseManifest`, `loadManifest`, `loadManifests`
- template helpers: `validateTemplate`, `parseTemplate`, `loadTemplate`, `injectArgs`
- cluster helpers: `connectHttp`, `startServer`, `createCluster`
- compose entrypoints: `composeDashboard`, `composeDashboardFromFiles`
- runtime types: `ComposeRequest`, `ComposeResult`, `DashboardTemplate`, `McpManifest`, `RuntimeErrorCode`, and related transport/cluster types

Important runtime behavior from the source:

- stdio servers are started with `--http --port=0` if those args are missing
- startup waits for a `listening on http://...` line on `stderr`
- tool calls go to `POST <baseUrl>/mcp`
- `composeDashboard()` resolves `ui://server/...` URIs into `${uiBaseUrl}/ui?uri=...`
- `composeDashboardFromFiles()` does not expose the `keepAlive` option that `composeDashboard()` supports

## `@casys/mcp-compose/deploy`

```typescript
import type {
  DeployCredentials,
  DeployManifestEntry,
  DeployRequest,
  DeployResult,
  DeployTransport,
  TunnelConnection,
} from "@casys/mcp-compose/deploy";
import { DeployErrorCode } from "@casys/mcp-compose/deploy";
```

Current state:

- `deploy` exports types and `DeployErrorCode`
- there are no deploy functions implemented or exported yet

The repo's deploy readme describes planned APIs such as `deployDashboard()` and `createTunnel()`, but those functions are not present in `0.3.0`.

## Key types

```typescript
interface CollectedUiResource {
  source: string;
  resourceUri: string;
  context?: Record<string, unknown>;
  slot: number;
}

interface UiOrchestration {
  layout: UiLayout;
  sync?: UiSyncRule[];
  sharedContext?: string[];
}

interface UiSyncRule {
  from: string;
  event: string;
  to: string;
  action: string;
}
```

For the data flow between those types, see [Collector](./collector/), [Composer](./composer/), [Renderer](./renderer/), and [Sync Rules](./sync-rules/).
