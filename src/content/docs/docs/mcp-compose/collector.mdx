## API

```typescript
import { createCollector, extractUiMeta } from "@casys/mcp-compose";
```

`createCollector()` returns a stateful `UiCollector`:

```typescript
interface UiCollector {
  collect(
    toolName: string,
    result: unknown,
    context?: Record<string, unknown>,
  ): CollectedUiResource | null;

  getResources(): CollectedUiResource[];
  clear(): void;
}
```

## `extractUiMeta(result)`

This is the low-level helper the collector uses internally. It only looks for `_meta.ui.resourceUri` and optional `visibility`.

```typescript
const meta = extractUiMeta({
  content: [{ type: "text", text: "OK" }],
  _meta: { ui: { resourceUri: "ui://pg/table/1", visibility: ["model", "app"] } },
});

// meta === { resourceUri: "ui://pg/table/1", visibility: ["model", "app"] }
```

If the input is not an object, `_meta` is missing, `ui` is missing, or `resourceUri` is empty or not a string, it returns `null`.

## `collect(toolName, result, context?)`

`collect()` wraps `extractUiMeta()` and stores a `CollectedUiResource` when a valid `resourceUri` exists:

```typescript
const collector = createCollector();

const resource = collector.collect(
  "postgres:query",
  {
    _meta: { ui: { resourceUri: "ui://pg/table/1" } },
  },
  { query: "SELECT * FROM users", userId: "u123" },
);

// resource === {
//   source: "postgres:query",
//   resourceUri: "ui://pg/table/1",
//   context: { query: "SELECT * FROM users", userId: "u123" },
//   slot: 0,
// }
```

Real behavior from the implementation and tests:

- Slot numbers are `0`-based and increase only when a resource is actually collected.
- Non-UI results are skipped and do not consume a slot.
- `context` is stored as-is and later used by `buildCompositeUi()` for shared context extraction.

Example:

```typescript
collector.collect("a", { _meta: { ui: { resourceUri: "ui://a" } } }); // slot 0
collector.collect("no-ui", { content: [] }); // null, no slot consumed
collector.collect("b", { _meta: { ui: { resourceUri: "ui://b" } } }); // slot 1
```

## `getResources()`

Returns the collected resources in slot order:

```typescript
const resources = collector.getResources();
```

This returns a new array copy. Mutating the returned array does not mutate the collector's internal state.

## `clear()`

Resets the collector:

```typescript
collector.clear();
```

After `clear()`, the next collected resource gets `slot: 0` again.

## Collected shape

```typescript
interface CollectedUiResource {
  source: string;
  resourceUri: string;
  context?: Record<string, unknown>;
  slot: number;
}
```

## SDK collector

If your inputs already match `@modelcontextprotocol/sdk` `CallToolResult` shape, use `createMcpSdkCollector()` from `@casys/mcp-compose/sdk`. It delegates to the same core collector and adds one behavior: `isError: true` results are skipped automatically.

### `McpSdkCollector` interface

```typescript
import { createMcpSdkCollector } from "@casys/mcp-compose/sdk";

interface McpSdkCollector {
  /**
   * Collect a UI resource from an MCP SDK CallToolResult.
   * Returns null if the result has isError: true or no _meta.ui.resourceUri.
   */
  collectFromSdk(
    toolName: string,
    result: McpSdkCallToolResult,
    context?: Record<string, unknown>,
  ): CollectedUiResource | null;

  /** Get all collected resources in slot order. */
  getResources(): CollectedUiResource[];

  /** Reset collected resources. */
  clear(): void;

  /** Access the underlying core collector. */
  readonly inner: UiCollector;
}
```

The method is `collectFromSdk()`, not `collect()`. The `inner` property exposes the underlying `UiCollector` if you need direct access to the core API.

### Example

```typescript
import { createMcpSdkCollector } from "@casys/mcp-compose/sdk";
import { buildCompositeUi, renderComposite } from "@casys/mcp-compose";

const collector = createMcpSdkCollector();

// sdkResult matches the @modelcontextprotocol/sdk CallToolResult shape
for (const [toolName, sdkResult] of toolCallResults) {
  collector.collectFromSdk(toolName, sdkResult, { workflowId: "wf-1" });
}

// Error results (isError: true) are automatically skipped
// Proceed with the standard pipeline
const resources = collector.getResources();
const descriptor = buildCompositeUi(resources, { layout: "split" });
const html = renderComposite(descriptor);
```

Note: `createMcpSdkCollector()` wraps a standard `createCollector()`. The `inner` property gives you access to that underlying collector if you need to mix SDK and non-SDK results.
