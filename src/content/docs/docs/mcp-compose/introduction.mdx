`mcp-compose` is a lightweight Deno library for taking several MCP tool results that expose `_meta.ui.resourceUri`, turning them into a single composition descriptor, and rendering a composite dashboard. These docs reflect package version `0.3.0`.

It is not a visual builder or end-user dashboard product. The repo README positions it as a composition primitive: you provide collected tool results plus orchestration, and it gives you HTML.

The public flow is:

```typescript
import {
  buildCompositeUi,
  createCollector,
  renderComposite,
} from "@casys/mcp-compose";

const collector = createCollector();
collector.collect("postgres:query", {
  _meta: { ui: { resourceUri: "ui://postgres/table/abc123" } },
});
collector.collect("viz:render", {
  _meta: { ui: { resourceUri: "ui://viz/chart/viz-1" } },
});

const descriptor = buildCompositeUi(collector.getResources(), {
  layout: "split",
  sync: [
    { from: "postgres:query", event: "filter", to: "viz:render", action: "update" },
  ],
});

const html = renderComposite(descriptor);
```

That maps directly to the code and tests in the repo:

- `createCollector()` inspects tool results and stores `CollectedUiResource` objects.
- `buildCompositeUi()` produces a `CompositeUiDescriptor` with resolved sync rules and optional shared context.
- `renderComposite()` turns that descriptor into a full HTML document with inline CSS and JS.

## What is implemented today

- The root package exports the collector, sync helpers, composer, host renderer, SDK helpers, and host types.
- `buildCompositeUi()` defaults to `layout: "stack"` when you omit orchestration.
- Invalid sync rules are not thrown from the composer. Unknown `from` or `to` references are silently excluded unless you validate first.
- The exported renderer comes from the `host` layer, not the older `core` renderer tree.
- The host renderer supports preset layouts (`split`, `tabs`, `grid`, `stack`) and object-based areas layouts.

## What each layer is for

- [`@casys/mcp-compose`](./api-reference/) is the default entrypoint.
- [`collector`](./collector/) covers `_meta.ui.resourceUri` extraction and slot assignment.
- [`composer`](./composer/) covers layout, shared context, and descriptor generation.
- [`renderer`](./renderer/) covers the generated HTML, CSS, and event bus behavior.
- [`sync-rules`](./sync-rules/) covers validation, resolution, and broadcast routing.

The repo also exposes `sdk`, `host`, `runtime`, and `deploy` subpaths. `deploy` is currently type-only: the types exist, but there are no deploy functions exported yet.
