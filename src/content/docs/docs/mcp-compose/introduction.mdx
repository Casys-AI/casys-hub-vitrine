## What is mcp-compose?

`@casys/mcp-compose` is a Deno library (JSR, v0.3.0) for assembling composite UIs from multiple MCP tool responses. When an MCP tool returns a `resourceUri` pointing to a micro-frontend, mcp-compose collects those references, composes them into a layout descriptor, and renders a complete self-contained HTML page that hosts each UI in a sandboxed iframe with a JSON-RPC 2.0 event bus.

It has zero dependencies, ships 219 tests, and is Deno-only.

## Design principles

**Zero dependencies.** The entire library uses only Deno built-ins (`crypto.randomUUID`, `TextEncoder`). No npm packages, no bundler required.

**Pure functions.** `buildCompositeUi` and `renderComposite` are stateless. Given the same inputs they always produce the same output. The only stateful object is the collector returned by `createCollector()`.

**Deterministic output.** Layout, slot ordering, and sync-rule resolution are all deterministic so the same tool invocations always produce the same composed page.

**Machine-readable errors.** No function in the public API throws a string. Validation results carry structured `ValidationIssue` objects with `ErrorCode` enum values and optional `path` fields. Callers can handle every error case programmatically.
