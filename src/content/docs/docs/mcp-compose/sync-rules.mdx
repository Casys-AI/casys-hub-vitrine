## `UiSyncRule`

Sync rules are authored with tool names:

```typescript
interface UiSyncRule {
  from: string;
  event: string;
  to: string;
  action: string;
}
```

Example from the repo tests:

```typescript
const rules = [
  { from: "postgres:query", event: "filter", to: "viz:render", action: "update" },
  { from: "date:picker", event: "change", to: "*", action: "refresh" },
];
```

`to: "*"` means broadcast to every other iframe.

## `validateSyncRules(rules, knownSources)`

```typescript
import { validateSyncRules } from "@casys/mcp-compose";
```

```typescript
const result = validateSyncRules(rules, ["postgres:query", "viz:render"]);
```

The validator checks two things:

- orphan references: `from` or `to` names that are not present in `knownSources`
- self-reference on non-broadcast rules: `from === to`

It returns:

```typescript
interface ValidationResult {
  valid: boolean;
  issues: ValidationIssue[];
}
```

```typescript
interface ValidationIssue {
  code: ErrorCode;
  message: string;
  path?: string;
}
```

The relevant error codes are:

- `ORPHAN_SYNC_REFERENCE`
- `CIRCULAR_SYNC_RULE`

Important: `CIRCULAR_SYNC_RULE` is only the direct self-reference case. The validator does not detect longer cycles such as `a -> b -> a`.

## `resolveSyncRules(rules, resources)`

```typescript
import { resolveSyncRules } from "@casys/mcp-compose";
```

This converts tool-name rules into slot-based rules:

```typescript
interface ResolvedSyncRule {
  from: number;
  event: string;
  to: number | "*";
  action: string;
}
```

`resolveSyncRules()` returns both `rules` and `issues`:

```typescript
const result = resolveSyncRules(
  [{ from: "postgres:query", event: "filter", to: "viz:render", action: "update" }],
  [
    { source: "postgres:query", resourceUri: "ui://pg/1", slot: 0 },
    { source: "viz:render", resourceUri: "ui://viz/2", slot: 1 },
  ],
);

// result.rules[0] === { from: 0, event: "filter", to: 1, action: "update" }
```

Unknown references are reported in `result.issues` and excluded from `result.rules`.

## Composer interaction

`buildCompositeUi()` calls `resolveSyncRules()` internally and keeps only `result.rules`. That means bad rules disappear unless you validate before composing.

Recommended pattern:

```typescript
const validation = validateSyncRules(
  orchestration.sync ?? [],
  resources.map((resource) => resource.source),
);

if (!validation.valid) {
  console.error(validation.issues);
  Deno.exit(1);
}

const descriptor = buildCompositeUi(resources, orchestration);
```

## Routing behavior in the renderer

<Mermaid chart={`sequenceDiagram
  participant UI1 as postgres:query (slot 0)
  participant BUS as Event Bus
  participant UI2 as viz:render (slot 1)
  UI1->>BUS: ui/update-model-context {event: "filter"}
  BUS->>BUS: Match sync rule: from=0, event=filter, to=1
  BUS->>UI2: ui/notifications/tool-result {action: "update", data, sharedContext}
  UI2-->>UI2: Apply filter update`} />

The generated host script matches rules by:

- source slot
- event name
- target slot or broadcast

Implementation note: the router treats `rule.event === "*"` as an event wildcard even though the public examples use concrete event names. That behavior is inferred from the host-renderer source.

## SDK-level validation

If your tools declare UI contracts with `uiMeta({ emits, accepts })`, use `validateComposition()` from `@casys/mcp-compose/sdk` in addition to core sync validation:

```typescript
import { uiMeta, validateComposition } from "@casys/mcp-compose/sdk";

const tools = [
  {
    name: "erp:customers",
    ...uiMeta({
      resourceUri: "ui://erp/customers",
      emits: ["rowSelected"],
    }),
  },
  {
    name: "viz:chart",
    ...uiMeta({
      resourceUri: "ui://viz/chart",
      accepts: ["highlight"],
    }),
  },
];

const result = validateComposition(tools, [
  { from: "erp:customers", event: "rowSelected", to: "viz:chart", action: "highlight" },
]);
```

`validateComposition()` adds semantic checks such as:

- `SYNC_EVENT_NOT_EMITTED`
- `SYNC_ACTION_NOT_ACCEPTED`
- `ORPHAN_EMIT`
- `ORPHAN_ACCEPT`
