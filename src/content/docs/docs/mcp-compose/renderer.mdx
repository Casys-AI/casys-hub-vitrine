## API

```typescript
import { renderComposite } from "@casys/mcp-compose";
```

The root export uses the host renderer from `@casys/mcp-compose/host`.

```typescript
function renderComposite(descriptor: CompositeUiDescriptor): string
```

## What it generates

`renderComposite()` returns one HTML document:

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>mcp-compose</title>
  <style>...</style>
</head>
<body>
  <div class="layout-split" id="container">
    <iframe id="ui-0" src="ui://postgres/table/123" data-slot="0" data-source="postgres:query"></iframe>
    <iframe id="ui-1" src="ui://viz/chart/456" data-slot="1" data-source="viz:chart"></iframe>
  </div>
  <script>...</script>
</body>
</html>
```

The renderer escapes tab labels and iframe attribute values before inserting them into the HTML.

## Layout behavior

The public renderer supports:

- `split`: `display: flex; height: 100vh`
- `tabs`: tab bar plus absolute-positioned iframes with an active first tab
- `grid`: `display: grid` with `repeat(auto-fit, minmax(400px, 1fr))`
- `stack`: vertical flex with `min-height: 200px` per iframe
- areas layouts: CSS `grid-template-areas` generated from `UiLayoutAreas`

Areas example:

```typescript
const descriptor = buildCompositeUi(resources, {
  layout: {
    areas: [
      ["filter", "main"],
      ["filter", "detail"],
    ],
    columns: [1, 3],
    rows: [2, 1],
    gap: "compact",
  },
});
```

When an areas layout is used, the renderer looks at `descriptor.areaMap` first. If there is no `areaMap`, it falls back to each child's `source` name as the area name.

## Event bus behavior

The inline script handles JSON-RPC 2.0 messages from the child UIs:

- `ui/initialize`: returns host capabilities and `hostContext.sharedContext`
- `ui/message`: logs a message and acknowledges it
- `ui/compose/event`: routes dedicated compose events using sync rules
- `ui/update-model-context`: legacy routing path, also using sync rules

Broadcast rules exclude the sender:

```typescript
{ from: "date:picker", event: "change", to: "*", action: "refresh" }
```

That rule fans out to every other iframe, not back to the source iframe.

## Shared context at render time

If the descriptor contains `sharedContext`, the renderer injects it into the script:

```typescript
const descriptor = buildCompositeUi(resources, {
  layout: "stack",
  sharedContext: ["workflowId"],
});
```

The initialize response includes that object under `hostContext.sharedContext`, and compose-event payloads also forward it to target iframes.

## Security and theming

Current host-renderer behavior in this repo:

- tool names are HTML-escaped before becoming tab labels
- `src`, `data-source`, and `data-area` are attribute-escaped
- base CSS defines light and dark custom properties
- `prefers-color-scheme: dark` is included

One implementation detail worth knowing: there is also an older core renderer in the repo, but the root package exports the host renderer. The host renderer is the one with areas-layout and `ui/compose/event` support.
