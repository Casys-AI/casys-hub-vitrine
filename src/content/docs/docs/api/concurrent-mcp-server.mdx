The main entry point for the framework. `ConcurrentMCPServer` handles tool registration, middleware composition, transport selection, and lifecycle management.

<Aside type="note">
  Deno-first. Node.js 22+ is supported via the JSR npm compatibility layer.
</Aside>

## Constructor

```typescript
import { ConcurrentMCPServer } from "@casys/mcp-server";

const server = new ConcurrentMCPServer(options: ConcurrentServerOptions);
```

### ConcurrentServerOptions

```typescript
interface ConcurrentServerOptions {
  // Required
  name: string;                          // Server name (shown to clients)
  version: string;                       // Semver version string

  // Concurrency
  maxConcurrent?: number;                // Default: 10
  backpressureStrategy?: "sleep" | "queue" | "reject";  // Default: "sleep"

  // Rate limiting
  rateLimit?: RateLimitOptions;

  // Schema validation
  validateSchema?: boolean;              // Default: false — enables ajv 8

  // Sampling
  enableSampling?: boolean;              // Enable bidirectional LLM delegation

  // Auth
  auth?: AuthOptions;                    // OAuth2/Bearer config

  // MCP Apps
  resourceCsp?: CspOptions;             // CSP options for HTML resource injection

  // Customization
  instructions?: string;                // Instructions sent to clients at initialization
  toolErrorMapper?: (error: unknown) => string;  // Custom error message formatter
  logger?: (msg: string) => void;       // Default: console.error
}
```

### RateLimitOptions

```typescript
interface RateLimitOptions {
  maxRequests: number;    // Max requests per window
  windowMs: number;       // Window size in milliseconds
}
```

### AuthOptions

```typescript
interface AuthOptions {
  provider: AuthProvider;   // Use presets (createGoogleAuthProvider, etc.) or JwtAuthProvider
}
```

## Registration methods

All registration must happen **before** calling `start()` or `startHttp()`.

### registerTool

```typescript
server.registerTool(tool: MCPTool, handler: ToolHandler): void;
```

Register a single tool with its handler:

```typescript
server.registerTool(
  {
    name: "greet",
    description: "Greet someone",
    inputSchema: {
      type: "object",
      properties: { name: { type: "string" } },
      required: ["name"],
    },
    requiredScopes: ["read"],   // Optional — requires this scope if auth is configured
    outputSchema: {             // Optional — for structured tool results (SDK 1.27)
      type: "object",
      properties: { message: { type: "string" } },
    },
    annotations: {              // Optional — hints for clients
      readOnlyHint: true,
    },
  },
  ({ name }) => `Hello, ${name}!`,
);
```

### registerTools

```typescript
server.registerTools(tools: MCPTool[], handlers: Map<string, ToolHandler>): void;
```

Bulk-register tools. Map keys must match tool names exactly.

### registerToolLive

```typescript
server.registerToolLive(tool: MCPTool, handler: ToolHandler): void;
```

Register a tool after the server has started. Sends a `tools/list_changed` notification to connected clients automatically.

### unregisterTool

```typescript
server.unregisterTool(toolName: string): void;
```

Remove a registered tool. Sends a `tools/list_changed` notification to connected clients.

### registerAppOnlyTool

```typescript
server.registerAppOnlyTool(tool: MCPTool, handler: ToolHandler): void;
```

Register a tool visible to UI clients only — it does not appear in the LLM's tool list.

### registerResource

```typescript
server.registerResource(resource: MCPResource, handler: ResourceHandler): void;
```

Register a single resource. See [MCP Apps](/docs/guides/mcp-apps/) for `text/html;profile=mcp-app` resources.

### registerResources

```typescript
server.registerResources(
  resources: MCPResource[],
  handlers: Map<string, ResourceHandler>,
): void;
```

Bulk-register resources.

### registerViewers

```typescript
server.registerViewers(config: ViewersConfig): void;
```

Bulk-register UI viewer resources from a configuration object:

```typescript
server.registerViewers({
  "ui://my-server/dashboard": {
    name: "Dashboard",
    description: "Metrics dashboard",
    handler: async (uri) => ({
      uri: uri.toString(),
      mimeType: MCP_APP_MIME_TYPE,
      text: renderDashboard(),
    }),
  },
});
```

### use

```typescript
server.use(middleware: Middleware): this;
```

Add a custom middleware to the pipeline. Returns `this` for chaining:

```typescript
server
  .use(logging)
  .use(metrics)
  .use(caching);
```

<Aside type="caution">
  Throws if called after `start()` or `startHttp()`. The pipeline is compiled at startup.
</Aside>

See the [Middleware Pipeline guide](/docs/guides/middleware/) for patterns and recipes.

## Transport methods

### start

```typescript
await server.start(): Promise<void>;
```

Start with STDIO transport. Use this for local tools (Claude Desktop, Cursor, etc.). Blocks until the client disconnects.

### startHttp

```typescript
await server.startHttp(options: HttpServerOptions): Promise<void>;
```

Start with Streamable HTTP transport. Includes SSE streaming, session management, and RFC 9728 discovery.

```typescript
interface HttpServerOptions {
  port: number;
  hostname?: string;              // Default: "0.0.0.0"
  cors?: boolean;                 // Default: true
  corsOrigins?: "*" | string[];   // Default: "*" — use allowlist in production
  maxBodyBytes?: number | null;   // Default: 1_000_000 (1 MB). null = no limit
  requireAuth?: boolean;          // Default: false — throws at startup if true without auth
  customRoutes?: RouteConfig[];   // Additional HTTP routes
  ipRateLimit?: {
    maxRequests: number;
    windowMs: number;
  };
}
```

HTTP routes served automatically:

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/mcp` | JSON-RPC tool calls |
| `GET` | `/mcp` | SSE stream for notifications |
| `GET` | `/health` | Health check |
| `GET` | `/metrics` | Prometheus metrics |
| `GET` | `/.well-known/oauth-protected-resource` | RFC 9728 discovery (if auth configured) |

Sessions: 32-character hex IDs, 30-minute TTL, max 10,000 active sessions, per-IP rate limit of 10 `initialize` calls per 60 seconds.

**Example with security options:**

```typescript
await server.startHttp({
  port: 3000,
  requireAuth: true,
  corsOrigins: ["https://app.example.com"],
  maxBodyBytes: 1_048_576,
  ipRateLimit: { maxRequests: 60, windowMs: 60_000 },
});
```

### stop

```typescript
await server.stop(): Promise<void>;
```

Graceful shutdown. Waits for in-flight requests to complete before closing.

## Notification methods

### sendNotification

```typescript
server.sendNotification(method: string, params?: Record<string, unknown>): void;
```

Send a notification to the current session (STDIO transport).

### broadcastNotification

```typescript
server.broadcastNotification(
  method: string,
  params?: Record<string, unknown>,
): void;
```

Push a notification to **all** connected SSE clients (HTTP transport).

### sendToSession

```typescript
server.sendToSession(sessionId: string, message: Record<string, unknown>): void;
```

Push a message to a specific SSE session by ID (HTTP transport).

## Observability methods

### getServerMetrics

```typescript
server.getServerMetrics(): ServerMetrics;
```

Returns current server metrics including in-flight requests, queued requests, and active sessions.

### getPrometheusMetrics

```typescript
server.getPrometheusMetrics(): string;
```

Returns metrics in Prometheus text format. Same data served at `GET /metrics`:

```
# HELP tool_calls_total Total number of tool calls
# TYPE tool_calls_total counter
tool_calls_total{tool="greet",status="success"} 42

# HELP tool_call_duration_ms Tool call duration in milliseconds
# TYPE tool_call_duration_ms histogram
tool_call_duration_ms_bucket{tool="greet",le="100"} 40
...

# HELP active_requests Currently active requests
# TYPE active_requests gauge
active_requests 3

# HELP active_sessions Currently active SSE sessions
# TYPE active_sessions gauge
active_sessions 7
```

Counters available: `tool_calls_total`, `auth_success_total`, `auth_failed_total`.

### getRateLimitMetrics

```typescript
server.getRateLimitMetrics(): { keys: number; totalRequests: number } | null;
```

Returns `null` if rate limiting is not configured.

### getToolCount / getToolNames

```typescript
server.getToolCount(): number;
server.getToolNames(): string[];
```

### getResourceCount

```typescript
server.getResourceCount(): number;
```

### isStarted

```typescript
server.isStarted(): boolean;
```

Returns `true` after `start()` or `startHttp()` has been called.

## Sampling bridge

When `enableSampling: true` is set, the server can delegate LLM calls back through the client connection:

```typescript
const server = new ConcurrentMCPServer({
  name: "my-server",
  version: "1.0.0",
  enableSampling: true,
});

server.registerTool(
  { name: "summarize", description: "Summarize text", inputSchema: { type: "object", properties: { text: { type: "string" } } } },
  async ({ text }, { bridge }) => {
    const result = await bridge.requestSampling({
      messages: [{ role: "user", content: text }],
      maxTokens: 500,
      maxIterations: 3,
    });
    return result.content;
  },
);
```

## OpenTelemetry

Set `OTEL_DENO=true` to enable OpenTelemetry tracing. Spans are emitted for tool calls and auth events.
