Factory functions for common identity providers, plus the low-level `JwtAuthProvider` for full control. All return an `AuthProvider` compatible with `McpAppOptions.auth.provider`.

## Factory functions (presets)

### createGoogleAuthProvider

```typescript
import { createGoogleAuthProvider } from "@casys/mcp-server";

const provider = createGoogleAuthProvider({
  audience: string;    // Your server's audience URL
  resource: string;    // Your server's resource URL
});
```

Uses Google's OIDC issuer (`https://accounts.google.com`) and public JWKS endpoint. JWKS URI is `https://www.googleapis.com/oauth2/v3/certs`.

### createAuth0AuthProvider

```typescript
import { createAuth0AuthProvider } from "@casys/mcp-server";

const provider = createAuth0AuthProvider({
  domain: string;              // Auth0 tenant domain (e.g., "my-tenant.auth0.com")
  audience: string;
  resource: string;
  scopesSupported?: string[];  // Scopes your server recognizes
});
```

### createGitHubAuthProvider

```typescript
import { createGitHubAuthProvider } from "@casys/mcp-server";

const provider = createGitHubAuthProvider({
  audience: string;
  resource: string;
});
```

For GitHub Actions OIDC tokens. Uses `https://token.actions.githubusercontent.com` as issuer.

### createOIDCAuthProvider

```typescript
import { createOIDCAuthProvider } from "@casys/mcp-server";

const provider = createOIDCAuthProvider({
  issuer: string;                      // OIDC issuer URL
  audience: string;
  resource: string;
  authorizationServers?: string[];     // Default: [issuer]
  scopesSupported?: string[];
});
```

Generic OIDC provider. If `jwksUri` is omitted, the JWKS URI defaults to `${issuer}/.well-known/jwks.json`. The provider does not fetch `.well-known/openid-configuration`.

<Aside type="tip">
  Not sure which preset to use? See the [Authentication guide](/docs/guides/auth) for usage examples with each provider.
</Aside>

## buildJwtAuthProvider

Public bridge function for building custom OIDC provider factories. Accepts raw strings, validates URLs, and picks the correct `JwtAuthProviderOptions` branch automatically:

```typescript
import { buildJwtAuthProvider, type BuildJwtAuthProviderOptions } from "@casys/mcp-server";

export function createKeycloakAuthProvider(options: {
  keycloakHost: string;
  realm: string;
  audience: string;
  resource: string;
}): JwtAuthProvider {
  const issuer = `https://${options.keycloakHost}/realms/${options.realm}`;
  return buildJwtAuthProvider({
    issuer,
    audience: options.audience,
    authorizationServers: [issuer],
    jwksUri: `${issuer}/protocol/openid-connect/certs`,
    resource: options.resource,
  }, "createKeycloakAuthProvider");
}
```

`BuildJwtAuthProviderOptions`:

```typescript
interface BuildJwtAuthProviderOptions {
  issuer: string;
  audience: string;
  jwksUri?: string;
  authorizationServers: string[];   // raw strings â€” bridge validates via httpsUrl()
  scopesSupported?: string[];
  resource: string;                 // HTTP(S) URL â†’ url branch; opaque URI â†’ opaque branch
  resourceMetadataUrl?: string;     // Required when resource is an opaque URI
}
```

## JwtAuthProvider

For full control over JWT validation. Use this when the presets don't cover your identity provider. Requires a `kind: "url" | "opaque"` discriminant tag:

```typescript
import { httpsUrl, JwtAuthProvider } from "@casys/mcp-server";

// URL resource branch (resource is an HTTP(S) URL â€” metadata URL auto-derived)
const provider = new JwtAuthProvider({
  kind: "url",
  issuer: "https://accounts.google.com",
  audience: "https://my-mcp.example.com",
  resource: httpsUrl("https://my-mcp.example.com"),
  authorizationServers: [httpsUrl("https://accounts.google.com")],
  jwksUri: "https://www.googleapis.com/oauth2/v3/certs",
  scopesSupported: ["read", "write"],
});

// Opaque resource branch (resource is an opaque URI â€” metadata URL required)
const provider = new JwtAuthProvider({
  kind: "opaque",
  issuer: "https://my-tenant.zitadel.cloud",
  audience: "367545125829670172",          // OIDC project ID as JWT aud
  resource: "367545125829670172",
  resourceMetadataUrl: httpsUrl("https://my-mcp.example.com/.well-known/oauth-protected-resource"),
  authorizationServers: [httpsUrl("https://my-tenant.zitadel.cloud")],
});
```

Most callers should use the preset factories or `buildJwtAuthProvider` instead of constructing `JwtAuthProvider` directly.

### verifyToken

```typescript
const authInfo = await provider.verifyToken(token: string);
// AuthInfo | null
```

Returns an `AuthInfo` object on success, `null` if the token is invalid or expired.

```typescript
interface AuthInfo {
  subject: string;                     // Token "sub" claim
  clientId?: string;                   // Token "client_id" or "azp" claim
  scopes: string[];                    // Parsed from "scope" claim (space-separated)
  claims?: Record<string, unknown>;    // All token claims
  expiresAt?: number;                  // Token expiry as Unix epoch seconds
  tenantId?: string;                   // Set by createMultiTenantMiddleware â€” undefined on single-tenant servers
}
```

### Token cache

`JwtAuthProvider` maintains an in-memory cache to avoid repeated JWKS roundtrips:

- Cache key: SHA-256 hash of the raw token string
- TTL: 5 minutes
- Max entries: 1000

## Config loader

For binary distribution â€” load auth config from YAML files and environment variables at runtime.

### AuthConfig (discriminated union)

`AuthConfig` is a discriminated union on `provider`. Each variant encodes its required fields at the type level:

```typescript
// Shared base fields
interface AuthConfigBase {
  audience: string;
  resource: string;
  scopesSupported?: string[];
  resourceMetadataUrl?: string;
}

interface GitHubAuthConfig extends AuthConfigBase { provider: "github"; }
interface GoogleAuthConfig extends AuthConfigBase { provider: "google"; }
interface Auth0AuthConfig  extends AuthConfigBase { provider: "auth0"; domain: string; }
interface OIDCAuthConfig   extends AuthConfigBase { provider: "oidc";  issuer: string; jwksUri?: string; }

type AuthConfig =
  | GitHubAuthConfig
  | GoogleAuthConfig
  | Auth0AuthConfig
  | OIDCAuthConfig;
```

TypeScript narrows on `config.provider`, so `config.domain` and `config.issuer` are typed as required (non-optional) in their respective branches â€” no runtime assertions needed.

### loadAuthConfig

```typescript
import { loadAuthConfig } from "@casys/mcp-server";

const config = await loadAuthConfig();
// AuthConfig | null
```

Auto-loads from `mcp-server.yaml` + `MCP_AUTH_*` env vars. Returns `null` if no configuration is found (auth stays disabled).

### createAuthProviderFromConfig

```typescript
import { createAuthProviderFromConfig } from "@casys/mcp-server";

const provider = createAuthProviderFromConfig(config: AuthConfig);
// Returns the appropriate AuthProvider based on config.provider
```

Dispatches to the correct preset based on `config.provider`:

| `config.provider` | Preset used |
|-------------------|-------------|
| `"google"` | `createGoogleAuthProvider` |
| `"auth0"` | `createAuth0AuthProvider` |
| `"github"` | `createGitHubAuthProvider` |
| `"oidc"` | `createOIDCAuthProvider` |

## Client OAuth

For building MCP clients that authenticate against protected servers. These utilities are not exported from the main `@casys/mcp-server` entry point â€” use the deep import path:

```typescript
import { prepareOAuthProvider } from "@casys/mcp-server/client-auth/connect";
import { FileTokenStore } from "@casys/mcp-server/client-auth/token-store/file-store";
import { MemoryTokenStore } from "@casys/mcp-server/client-auth/token-store/memory-store";

const { provider, callbackServer } = await prepareOAuthProvider(
  "https://my-mcp.example.com",
  {
    clientId: "my-client-id",
    scopes: ["read", "write"],
    tokenStore: new FileTokenStore("./.tokens"),  // or new MemoryTokenStore()
    openBrowser: async (url) => {
      // Open the authorization URL in the user's browser
      console.log(`Open this URL to authorize: ${url}`);
    },
  },
);
// Pass provider as authProvider to StreamableHTTPClientTransport
// Close callbackServer when done
```

`FileTokenStore(path)` persists tokens to disk between restarts. `MemoryTokenStore()` is in-memory only.
