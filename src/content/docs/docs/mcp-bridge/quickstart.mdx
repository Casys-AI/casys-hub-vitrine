## Prerequisites

- Deno 2.x or Node.js 20+
- A Telegram bot token from [@BotFather](https://t.me/botfather)
- An MCP server reachable over HTTP JSON-RPC
- An HTML app directory you want to expose through `assetDirectories`

## Install

```bash
# npm
npm install @casys/mcp-bridge

# Deno
deno add jsr:@casys/mcp-bridge
```

<Aside type="caution">
  The injected runtime is served as `/bridge.js?...` and always opens `/bridge?session=...` at the same origin root. If you put the bridge behind a reverse proxy, make sure `/app`, `/bridge.js`, `/bridge`, and optionally `/ui` are reachable at the root of that origin.
</Aside>

## Start the bridge

<Steps>

1. **Create the Resource Server**

   ```ts
   import { JsonRpcMcpBackend, startResourceServer } from "@casys/mcp-bridge";

   const server = startResourceServer({
     assetDirectories: {
       "my-app": "./ui-dist",
     },
     platform: "telegram",
     telegramBotToken: Deno.env.get("TELEGRAM_BOT_TOKEN")!,
     backend: new JsonRpcMcpBackend({
       endpointUrl: "https://my-mcp.example.com/mcp",
     }),
     options: {
       resourceServerPort: 4000,
     },
   });

   console.log(`Bridge running at ${server.baseUrl}`);
   ```

2. **Publish the required routes**

   Your public origin needs these paths:

   - `/app/*` for locally served HTML, JS, CSS, images, and fonts
   - `/bridge.js` for the injected browser runtime
   - `/bridge` for the WebSocket
   - `/ui` only if you want backend-proxied `ui://` resources

3. **Point Telegram at the HTML page**

   Set the Mini App URL in [@BotFather](https://t.me/botfather) to the page served by the bridge, for example:

   ```text
   https://your-domain.com/app/my-app/index.html
   ```

4. **Open the Mini App**

   When Telegram loads the page, the Resource Server creates a session, injects `bridge.js`, and the browser runtime sends Telegram auth data over the WebSocket before normal JSON-RPC traffic begins.

</Steps>

## Optional: observe bridge readiness

The injected runtime emits browser events after the socket and optional auth step complete:

```ts
window.addEventListener("mcp-bridge-ready", (event) => {
  console.log("Bridge ready", event.detail);
});

window.addEventListener("mcp-bridge-auth-error", (event) => {
  console.error("Bridge auth failed", event.detail.error);
});
```

## Local assets vs backend `ui://` resources

- Use `/app/<server>/<path>` when the HTML lives in `assetDirectories`.
- Use `/ui?uri=ui://...` when your configured backend implements `readResource()`.
- `JsonRpcMcpBackend.readResource()` forwards a `resources/read` JSON-RPC request and uses only the first `contents[0].text` value as HTML.
