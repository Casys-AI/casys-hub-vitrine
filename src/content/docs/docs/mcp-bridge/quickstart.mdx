## Prerequisites

- [Deno](https://deno.land) 2.x, or Node.js 20+ (Web Crypto required)
- A Telegram Bot token (from [@BotFather](https://t.me/botfather))
- An existing MCP Server with `ui://` resources

## Installation

```bash
# Deno (JSR)
deno add jsr:@casys/mcp-bridge

# Node.js (via JSR on npm)
npx jsr add @casys/mcp-bridge
```

## Setup

<Steps>

1. **Create the bridge server**

   ```typescript
   // bridge.ts
   import { startResourceServer, JsonRpcMcpBackend } from "@casys/mcp-bridge";

   // Connect to your existing MCP server
   const backend = new JsonRpcMcpBackend({ /* your MCP server config */ });

   startResourceServer({
     assetDirectories: { "my-app": "./ui-dist" },
     platform: "telegram",
     telegramBotToken: Deno.env.get("TELEGRAM_BOT_TOKEN")!,
     backend,
     options: {
       resourceServerPort: 4000,
       allowedOrigins: ["https://your-domain.com"],
       debug: true,
     },
   });
   ```

   Run it:

   ```bash
   deno run --allow-net --allow-read --allow-env bridge.ts
   ```

2. **Expose your server publicly**

   For development, use a tunnel:

   ```bash
   ngrok http 4000
   ```

   Note the HTTPS URL — you'll need it for BotFather.

3. **Configure the Telegram Mini App via @BotFather**

   In Telegram, open [@BotFather](https://t.me/botfather) and run `/setmenubutton`. Select your bot and set the URL to your public HTTPS URL (e.g., `https://abc123.ngrok.io`).

   The bridge serves your MCP App at `GET /ui?uri=ui://my-app/index.html`.

4. **Open the Mini App**

   Open your bot in Telegram and tap the menu button. The bridge:
   - Creates a session via `POST /session` (with Telegram `initData`)
   - Validates `initData` HMAC-SHA256 using your bot token
   - Serves your HTML with `bridge.js` injected
   - Opens a WebSocket at `WS /bridge?session=<id>`
   - Forwards all `postMessage` JSON-RPC calls to your MCP Server

</Steps>

<Aside type="tip">
  Theme synchronization is automatic. When the user switches Telegram to dark mode, `TelegramPlatformAdapter` maps `themeParams` to CSS custom properties and sends a `ui/notifications/host-context-changed` notification to your app.
</Aside>

## Message flow

```
Telegram WebView
  -> POST /session (initData validated via HMAC-SHA256)
  -> GET /ui?uri=ui://... (HTML + injected bridge.js)
  -> WS /bridge?session=id (auth as first WS message)
  -> postMessage({ jsonrpc: "2.0", method: "tools/call", ... })
  -> bridge.js intercepts -> sends via WebSocket
  -> Resource Server -> backend.handleMessage()
  -> MCP Server executes tool
  -> result flows back via WebSocket -> window.dispatchEvent(MessageEvent)
```

## Tool Result Push Pattern

If your MCP tool produces a result you want pre-loaded in the Mini App before it opens:

```typescript
import { storeToolResult } from "@casys/mcp-bridge";

const ref = storeToolResult(toolOutputData);
// Include ?ref=<ref> in the Mini App URL.
// bridge.js will auto-attach it as ui/notifications/tool-result on session connect.
```
