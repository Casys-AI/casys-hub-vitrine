<Aside type="caution">
  The LINE adapter is partial. `LineAdapter` and `createLineRuntime()` are available, but there is no built-in server-side auth handler. You must implement your own LINE token validation.
</Aside>

## What is implemented

- `LineAdapter` — extends `BasePostMessageAdapter`, wires up LIFF SDK for transport
- `createLineRuntime()` — creates a runtime using `liff.getAccessToken()` for auth token retrieval
- `LiffSdk` interface — type definition for the LIFF SDK object

## What is not implemented

- A `BridgeAuthHandler` for server-side LINE token verification
- Built-in LINE OAuth token validation against LINE's API
- Automatic session creation with LINE auth (unlike the Telegram flow, nothing happens automatically)

## Setup

### Client-side

`LineAdapter` expects a LIFF SDK instance conforming to the `LiffSdk` interface:

```typescript
import { LineAdapter, createLineRuntime } from "@casys/mcp-bridge";

// liff is the LINE LIFF SDK, loaded from https://static.line-scdn.net/liff/edge/2/sdk.js
const adapter = new LineAdapter(liff);

// Or use createLineRuntime for a higher-level wrapper
const runtime = createLineRuntime(liff);
```

The `LiffSdk` interface:

```typescript
interface LiffSdk {
  init(config: { liffId: string }): Promise<void>;
  isLoggedIn(): boolean;
  getAccessToken(): string | null;
  sendMessages(messages: object[]): Promise<void>;
  closeWindow(): void;
}
```

`createLineRuntime()` calls `liff.getAccessToken()` and includes the token in the auth payload sent to the Resource Server on WebSocket connect.

### Server-side auth (you must implement this)

Since there is no built-in LINE auth handler, you need to provide one via the `auth` option:

```typescript
import { startResourceServer } from "@casys/mcp-bridge";

startResourceServer({
  assetDirectories: { "line-app": "./ui-dist" },
  platform: "line",
  auth: async (session, authPayload) => {
    // authPayload.token is the LINE access token from liff.getAccessToken()
    const response = await fetch("https://api.line.me/oauth2/v2.1/verify", {
      method: "POST",
      headers: { "Content-Type": "application/x-www-form-urlencoded" },
      body: `access_token=${authPayload.token}`,
    });
    if (!response.ok) throw new Error("Invalid LINE token");
    const data = await response.json();
    return { userId: data.sub };
  },
  options: { resourceServerPort: 4000 },
});
```

<Aside type="note">
  The `auth` parameter accepts a `BridgeAuthHandler` function. It is called with the session object and the auth payload sent by the client as the first WebSocket message.
</Aside>

## LIFF setup

1. Register your app at the [LINE Developers Console](https://developers.line.biz/console/)
2. Create a LIFF app with your Resource Server URL as the endpoint
3. Set the LIFF ID in your client-side initialization:

```javascript
await liff.init({ liffId: "YOUR_LIFF_ID" });
```
