## `startResourceServer(config)`

`startResourceServer()` is synchronous and returns a running server object immediately.

```ts
import { JsonRpcMcpBackend, startResourceServer } from "@casys/mcp-bridge";

const server = startResourceServer({
  assetDirectories: {
    "my-app": "./ui-dist",
  },
  platform: "telegram",
  telegramBotToken: Deno.env.get("TELEGRAM_BOT_TOKEN")!,
  backend: new JsonRpcMcpBackend({
    endpointUrl: "https://my-mcp.example.com/mcp",
  }),
  options: {
    resourceServerPort: 4000,
    debug: true,
  },
  csp: {
    frameAncestors: ["https://web.telegram.org"],
  },
});

console.log(server.baseUrl);
```

## `ResourceServerConfig`

| Field | Type | Notes |
| --- | --- | --- |
| `assetDirectories` | `Record<string, string>` | Required. Maps the `ui://` server name to a local directory for `/app/<server>/...`. |
| `platform` | `string` | Required. Used in injected `/bridge.js?...` query parameters. |
| `options` | `BridgeOptions` | `resourceServerPort`, `allowedOrigins`, and `debug`. |
| `csp` | `CspOptions` | Extra CSP sources and `allowInline` behavior. |
| `auth` | `BridgeAuthHandler` | Optional generic auth hook for WebSocket auth-on-first-message. |
| `telegramBotToken` | `string` | Convenience path for Telegram auth. Required for `platform: "telegram"` unless you pass `auth`. |
| `backend` | `McpBackend` | Handles normal JSON-RPC and optional `readResource()` calls. |
| `uiPath` | `string \| null` | Built-in backend UI route. Defaults to `"/ui"`. Set `null` to disable it. |
| `onMessage` | `(session, message) => Promise<McpAppsMessage \| null>` | Runs before `backend.handleMessage()`. |
| `onHttpRequest` | `(request) => Promise<Response \| { html: string; pendingNotifications?: PendingNotification[] } \| null>` | Custom HTTP hook for non-built-in routes. |

## Built-in routes

The server exposes:

- `GET /health`
- `POST /session`
- `GET /bridge.js`
- `GET/WS /bridge?session=<id>`
- `GET /app/<server>/<path>`
- `GET /ui?uri=ui://...` when `uiPath` is enabled and the backend has `readResource()`

## Return value

```ts
interface ResourceServer {
  readonly baseUrl: string;
  readonly sessions: SessionStore;
  storeToolResult(result: ToolResultData): string;
  consumeToolResult(ref: string): ToolResultData | undefined;
  stop(): Promise<void>;
}
```

## Tool-result handoff

`storeToolResult()` lets you stage a `ui/notifications/tool-result` notification before the WebSocket is open.

```ts
const ref = server.storeToolResult({
  content: [{ type: "text", text: "done" }],
});

const url = `https://your-domain.com/some-page?ref=${ref}`;
```

If that page is served through proxied HTML, the server consumes the stored result, buffers the notification on the session, and flushes it when the socket opens. Stored results are single-use and expire after 5 minutes.

## `JsonRpcMcpBackend`

`JsonRpcMcpBackend` is the built-in generic backend for HTTP JSON-RPC MCP servers.

```ts
import { JsonRpcMcpBackend } from "@casys/mcp-bridge";

const backend = new JsonRpcMcpBackend({
  endpointUrl: "https://my-mcp.example.com/mcp",
  headers: {
    Authorization: "Bearer token",
  },
});
```

### `JsonRpcMcpBackendOptions`

| Field | Type | Notes |
| --- | --- | --- |
| `endpointUrl` | `string` | Required. Full HTTP URL accepting JSON-RPC POST requests. Trailing slash is stripped. |
| `headers` | `Record<string, string>` | Optional. Static headers added to every request (e.g. `Authorization`). |
| `fetchFn` | `typeof fetch` | Optional. Custom fetch implementation. Useful for tests or custom runtime environments. Defaults to global `fetch`. |

It:

- POSTs JSON-RPC messages to `endpointUrl`
- turns transport failures into JSON-RPC `-32603` errors for request messages
- implements `readResource()` by sending a `resources/read` request
- returns only the first `contents[0].text` field as HTML
