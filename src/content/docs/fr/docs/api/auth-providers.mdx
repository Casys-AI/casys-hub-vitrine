Fonctions factory pour les fournisseurs d'identite courants, plus le `JwtAuthProvider` bas niveau pour un controle total. Toutes retournent un `AuthProvider` compatible avec `McpAppOptions.auth.provider`.

## Fonctions factory (presets)

### createGoogleAuthProvider

```typescript
import { createGoogleAuthProvider } from "@casys/mcp-server";

const provider = createGoogleAuthProvider({
  audience: string;    // Your server's audience URL
  resource: string;    // Your server's resource URL
});
```

Utilise l'issuer OIDC de Google (`https://accounts.google.com`) et l'endpoint JWKS public. L'URI JWKS est `https://www.googleapis.com/oauth2/v3/certs`.

### createAuth0AuthProvider

```typescript
import { createAuth0AuthProvider } from "@casys/mcp-server";

const provider = createAuth0AuthProvider({
  domain: string;              // Auth0 tenant domain (e.g., "my-tenant.auth0.com")
  audience: string;
  resource: string;
  scopesSupported?: string[];  // Scopes your server recognizes
});
```

### createGitHubAuthProvider

```typescript
import { createGitHubAuthProvider } from "@casys/mcp-server";

const provider = createGitHubAuthProvider({
  audience: string;
  resource: string;
});
```

Pour les tokens OIDC de GitHub Actions. Utilise `https://token.actions.githubusercontent.com` comme issuer.

### createOIDCAuthProvider

```typescript
import { createOIDCAuthProvider } from "@casys/mcp-server";

const provider = createOIDCAuthProvider({
  issuer: string;                      // OIDC issuer URL
  audience: string;
  resource: string;
  authorizationServers?: string[];     // Default: [issuer]
  scopesSupported?: string[];
});
```

Fournisseur OIDC generique. Si `jwksUri` est omis, l'URI JWKS prend par defaut la valeur `${issuer}/.well-known/jwks.json`. Le fournisseur ne recupere pas `.well-known/openid-configuration`.

<Aside type="tip">
  Pas sur de quel preset utiliser ? Consultez le [Guide d'authentification](/fr/docs/guides/auth/) pour des exemples d'utilisation avec chaque fournisseur.
</Aside>

## buildJwtAuthProvider

Fonction bridge publique pour construire des factories de fournisseurs OIDC personnalises. Accepte des chaines brutes, valide les URLs, et choisit automatiquement la bonne branche de `JwtAuthProviderOptions` :

```typescript
import { buildJwtAuthProvider, type BuildJwtAuthProviderOptions } from "@casys/mcp-server";

export function createKeycloakAuthProvider(options: {
  keycloakHost: string;
  realm: string;
  audience: string;
  resource: string;
}): JwtAuthProvider {
  const issuer = `https://${options.keycloakHost}/realms/${options.realm}`;
  return buildJwtAuthProvider({
    issuer,
    audience: options.audience,
    authorizationServers: [issuer],
    jwksUri: `${issuer}/protocol/openid-connect/certs`,
    resource: options.resource,
  }, "createKeycloakAuthProvider");
}
```

`BuildJwtAuthProviderOptions` :

```typescript
interface BuildJwtAuthProviderOptions {
  issuer: string;
  audience: string;
  jwksUri?: string;
  authorizationServers: string[];   // chaines brutes — le bridge valide via httpsUrl()
  scopesSupported?: string[];
  resource: string;                 // URL HTTP(S) → branche url ; URI opaque → branche opaque
  resourceMetadataUrl?: string;     // Requis quand resource est un URI opaque
}
```

## JwtAuthProvider

Pour un controle total sur la validation JWT. A utiliser quand les presets ne couvrent pas votre fournisseur d'identite. Requiert un tag discriminant `kind: "url" | "opaque"` :

```typescript
import { httpsUrl, JwtAuthProvider } from "@casys/mcp-server";

// Branche URL resource (resource est une URL HTTP(S) — metadata URL auto-derivee)
const provider = new JwtAuthProvider({
  kind: "url",
  issuer: "https://accounts.google.com",
  audience: "https://my-mcp.example.com",
  resource: httpsUrl("https://my-mcp.example.com"),
  authorizationServers: [httpsUrl("https://accounts.google.com")],
  jwksUri: "https://www.googleapis.com/oauth2/v3/certs",
  scopesSupported: ["read", "write"],
});

// Branche opaque resource (resource est un URI opaque — metadata URL requise)
const provider = new JwtAuthProvider({
  kind: "opaque",
  issuer: "https://my-tenant.zitadel.cloud",
  audience: "367545125829670172",
  resource: "367545125829670172",
  resourceMetadataUrl: httpsUrl("https://my-mcp.example.com/.well-known/oauth-protected-resource"),
  authorizationServers: [httpsUrl("https://my-tenant.zitadel.cloud")],
});
```

Dans la plupart des cas, preferez les factories preset ou `buildJwtAuthProvider` plutot que de construire `JwtAuthProvider` directement.

### verifyToken

```typescript
const authInfo = await provider.verifyToken(token: string);
// AuthInfo | null
```

Retourne un objet `AuthInfo` en cas de succes, `null` si le token est invalide ou expire.

```typescript
interface AuthInfo {
  subject: string;                     // Token "sub" claim
  clientId?: string;                   // Token "client_id" or "azp" claim
  scopes: string[];                    // Parsed from "scope" claim (space-separated)
  claims?: Record<string, unknown>;    // All token claims
  expiresAt?: number;                  // Token expiry as Unix epoch seconds
  tenantId?: string;                   // Set by createMultiTenantMiddleware — undefined on single-tenant servers
}
```

### Cache de tokens

`JwtAuthProvider` maintient un cache en memoire pour eviter des roundtrips JWKS repetes :

- Cle de cache : hash SHA-256 de la chaine de token brute
- TTL : 5 minutes
- Entrees max : 1000

## Chargeur de config

Pour la distribution binaire — charge la config d'auth depuis des fichiers YAML et des variables d'environnement au runtime.

### AuthConfig (union discriminee)

`AuthConfig` est une union discriminee sur `provider`. Chaque variante encode ses champs requis au niveau du type :

```typescript
interface GitHubAuthConfig extends AuthConfigBase { provider: "github"; }
interface GoogleAuthConfig extends AuthConfigBase { provider: "google"; }
interface Auth0AuthConfig  extends AuthConfigBase { provider: "auth0"; domain: string; }
interface OIDCAuthConfig   extends AuthConfigBase { provider: "oidc";  issuer: string; jwksUri?: string; }

type AuthConfig =
  | GitHubAuthConfig
  | GoogleAuthConfig
  | Auth0AuthConfig
  | OIDCAuthConfig;
```

TypeScript affine sur `config.provider`, donc `config.domain` et `config.issuer` sont types comme requis dans leurs branches respectives.

### loadAuthConfig

```typescript
import { loadAuthConfig } from "@casys/mcp-server";

const config = await loadAuthConfig();
// AuthConfig | null
```

Charge automatiquement depuis `mcp-server.yaml` + variables `MCP_AUTH_*`. Retourne `null` si aucune configuration n'est trouvee (l'auth reste desactivee).

### createAuthProviderFromConfig

```typescript
import { createAuthProviderFromConfig } from "@casys/mcp-server";

const provider = createAuthProviderFromConfig(config: AuthConfig);
// Returns the appropriate AuthProvider based on config.provider
```

Dispatche vers le bon preset selon `config.provider` :

| `config.provider` | Preset utilise |
|-------------------|----------------|
| `"google"` | `createGoogleAuthProvider` |
| `"auth0"` | `createAuth0AuthProvider` |
| `"github"` | `createGitHubAuthProvider` |
| `"oidc"` | `createOIDCAuthProvider` |

## Client OAuth

Pour construire des clients MCP qui s'authentifient contre des serveurs proteges. Ces utilitaires ne sont pas exportes depuis le point d'entree principal `@casys/mcp-server` — utilisez le chemin d'import profond :

```typescript
import { prepareOAuthProvider } from "@casys/mcp-server/client-auth/connect";
import { FileTokenStore } from "@casys/mcp-server/client-auth/token-store/file-store";
import { MemoryTokenStore } from "@casys/mcp-server/client-auth/token-store/memory-store";

const { provider, callbackServer } = await prepareOAuthProvider(
  "https://my-mcp.example.com",
  {
    clientId: "my-client-id",
    scopes: ["read", "write"],
    tokenStore: new FileTokenStore("./.tokens"),  // or new MemoryTokenStore()
    openBrowser: async (url) => {
      // Open the authorization URL in the user's browser
      console.log(`Open this URL to authorize: ${url}`);
    },
  },
);
// Pass provider as authProvider to StreamableHTTPClientTransport
// Close callbackServer when done
```

`FileTokenStore(path)` persiste les tokens sur disque entre les redemarrages. `MemoryTokenStore()` est uniquement en memoire.
