La MCP Platform regroupe trois packages qui fonctionnent ensemble : **construire** des serveurs MCP avec des outils produisant des interfaces, **les connecter** aux plateformes de messagerie, et **composer** plusieurs interfaces en un seul tableau de bord.

## Les trois packages

<CardGrid>
  <Card title="@casys/mcp-server" icon="rocket">
    Un framework pour construire des serveurs MCP sur Deno. Pipeline de middleware, OAuth2, controle de concurrence, et `McpApp` — la classe qui enregistre les outils et sert les ressources `ui://`.
  </Card>
  <Card title="@casys/mcp-bridge" icon="laptop">
    Relie les interfaces des MCP Apps aux plateformes de messagerie (Telegram, LINE). Execute un serveur de ressources HTTP qui sert des pages HTML dans les webviews des plateformes via WebSocket JSON-RPC.
  </Card>
  <Card title="@casys/mcp-compose" icon="list-format">
    Collecte les resultats `_meta.ui` de plusieurs appels d'outils et les compose en un tableau de bord HTML a panneaux divises. Un bus d'evenements permet aux iframes de communiquer.
  </Card>
</CardGrid>

## Le contrat `_meta.ui`

Les trois packages se connectent via une convention partagee : lorsqu'un outil retourne un resultat, il inclut un champ `_meta.ui.resourceUri`. La valeur est un URI `ui://` qui pointe vers une page HTML enregistree sur le serveur.

```typescript
// Un resultat d'outil qui porte une ressource UI
{
  content: [{ type: "text", text: "Invoice INV-001 loaded" }],
  _meta: { ui: { resourceUri: "ui://invoices/invoice-viewer" } }
}
```

`mcp-server` enregistre le HTML a cet URI. `mcp-bridge` resout l'URI et sert le HTML dans une webview Telegram Mini App. `mcp-compose` collecte plusieurs de ces resultats et les affiche cote a cote dans un tableau de bord d'iframes.

## Quel package utiliser

**Utilisez `mcp-server` seul** quand vous avez besoin d'un serveur MCP — pour tout cas d'usage impliquant la creation d'outils. Il ne depend pas de `mcp-bridge` ni de `mcp-compose`.

**Ajoutez `mcp-bridge`** quand vos utilisateurs interagissent avec le serveur via une plateforme de messagerie (Telegram, LINE) et que vous souhaitez ouvrir des pages HTML interactives dans les webviews de la plateforme.

**Ajoutez `mcp-compose`** quand un workflow LLM appelle plusieurs outils retournant chacun une interface et que vous voulez afficher ces interfaces cote a cote dans un seul tableau de bord. `mcp-compose` peut etre utilise sans `mcp-bridge` — il produit du HTML brut.

**Utilisez les trois** quand vous voulez un pipeline complet : outils MCP avec interfaces, delivres dans une plateforme de messagerie, avec communication d'evenements entre interfaces.
