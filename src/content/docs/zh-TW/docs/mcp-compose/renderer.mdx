## API

```typescript
import { renderComposite } from "@casys/mcp-compose";
```

根匯出使用來自 `@casys/mcp-compose/host` 的主機渲染器。

```typescript
function renderComposite(descriptor: CompositeUiDescriptor): string
```

## 生成的內容

`renderComposite()` 返回一個 HTML 文件：

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>mcp-compose</title>
  <style>...</style>
</head>
<body>
  <div class="layout-split" id="container">
    <iframe id="ui-0" src="ui://postgres/table/123" data-slot="0" data-source="postgres:query"></iframe>
    <iframe id="ui-1" src="ui://viz/chart/456" data-slot="1" data-source="viz:chart"></iframe>
  </div>
  <script>...</script>
</body>
</html>
```

渲染器在將標籤和 iframe 屬性值插入 HTML 之前會進行跳脫處理。

## 版面配置行為

公開的渲染器支援：

- `split`：`display: flex; height: 100vh`
- `tabs`：標籤列加上絕對定位的 iframe，預設啟用第一個標籤
- `grid`：`display: grid`，使用 `repeat(auto-fit, minmax(400px, 1fr))`
- `stack`：垂直 flex，每個 iframe 最小高度 `min-height: 200px`
- 區域版面配置：從 `UiLayoutAreas` 生成的 CSS `grid-template-areas`

區域範例：

```typescript
const descriptor = buildCompositeUi(resources, {
  layout: {
    areas: [
      ["filter", "main"],
      ["filter", "detail"],
    ],
    columns: [1, 3],
    rows: [2, 1],
    gap: "compact",
  },
});
```

當使用區域版面配置時，渲染器首先查看 `descriptor.areaMap`。如果沒有 `areaMap`，則退回使用每個子項的 `source` 名稱作為區域名稱。

## 事件匯流排行為

內嵌腳本處理來自子 UI 的 JSON-RPC 2.0 訊息：

- `ui/initialize`：返回主機能力和 `hostContext.sharedContext`
- `ui/message`：記錄訊息並確認收到
- `ui/compose/event`：使用同步規則路由專用的組合事件
- `ui/update-model-context`：傳統路由路徑，也使用同步規則

廣播規則排除發送者：

```typescript
{ from: "date:picker", event: "change", to: "*", action: "refresh" }
```

該規則會廣播到每個其他 iframe，而不會回傳給來源 iframe。

## 渲染時的共享上下文

如果描述符包含 `sharedContext`，渲染器會將其注入腳本中：

```typescript
const descriptor = buildCompositeUi(resources, {
  layout: "stack",
  sharedContext: ["workflowId"],
});
```

初始化回應會將該物件包含在 `hostContext.sharedContext` 下，組合事件負載也會將其轉發給目標 iframe。

## 安全性與主題設定

此套件庫中目前的主機渲染器行為：

- 工具名稱在成為標籤文字之前會進行 HTML 跳脫
- `src`、`data-source` 和 `data-area` 會進行屬性跳脫
- 基礎 CSS 定義了淺色和深色自訂屬性
- 包含 `prefers-color-scheme: dark`

一個值得注意的實作細節：套件庫中也有一個較舊的核心渲染器，但根套件匯出的是主機渲染器。主機渲染器才是支援區域版面配置和 `ui/compose/event` 的版本。
